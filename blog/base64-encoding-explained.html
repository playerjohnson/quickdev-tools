Base64 Encoding Explained: When and Why to Use It

14 February 2026 · 6 min read

What Is Base64?

Base64 is a binary-to-text encoding scheme that converts binary data into a string of 64 ASCII characters (A-Z, a-z, 0-9, + and /). It's not encryption — it's encoding. Anyone can decode Base64 back to the original data. Its purpose is to safely transmit binary data through systems that only handle text.

Every 3 bytes of input produce 4 characters of Base64 output, so encoded data is roughly 33% larger than the original. The = character is used as padding when the input length isn't divisible by 3.

How It Works

The encoding process takes three bytes (24 bits) and splits them into four 6-bit groups. Each 6-bit value maps to one of 64 characters:

Input:  "Hi" → 01001000 01101001
Split:  010010 000110 1001xx
Padded: 010010 000110 100100
Base64: S      G      k      =
Result: "SGk="

Common Use Cases

Data URIs in HTML/CSS: Embed small images directly in your code without a separate HTTP request. Useful for icons under ~2KB.

<img src="data:image/png;base64,iVBORw0KGgo..." />

background-image: url(data:image/svg+xml;base64,PHN2Zy...);

HTTP Basic Authentication: The Authorization header sends credentials as Base64-encoded username:password.

Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=
// Decodes to: username:password

This is why Basic Auth must always use HTTPS — Base64 is trivially decoded.

JWT Tokens: JSON Web Tokens use Base64URL encoding (replacing + with - and / with _) for the header and payload sections.

Email Attachments (MIME): Binary attachments in emails are Base64-encoded because SMTP only supports 7-bit ASCII text.

API Request Bodies: When sending binary data (images, files) in JSON payloads, Base64 encoding lets you include them as string values.

When NOT to Use Base64

  • Storing passwords — Base64 is not encryption. Use bcrypt, scrypt or Argon2 for password hashing.
  • Large files in HTML — Data URIs for images over ~5KB actually hurt performance. Use separate files.
  • Sensitive data in URLs — Base64URL doesn't hide data. Use proper encryption.
  • When binary transport is available — multipart/form-data, WebSockets binary frames and gRPC handle binary natively without the 33% size overhead.

Base64 in Every Language

// JavaScript (browser)
btoa("Hello World")        // "SGVsbG8gV29ybGQ="
atob("SGVsbG8gV29ybGQ=")   // "Hello World"

// JavaScript (Node.js)
Buffer.from("Hello").toString("base64")
Buffer.from("SGVsbG8=", "base64").toString()

// Python
import base64
base64.b64encode(b"Hello")  # b'SGVsbG8='
base64.b64decode(b"SGVsbG8=")

// C#
Convert.ToBase64String(Encoding.UTF8.GetBytes("Hello"))
Encoding.UTF8.GetString(Convert.FromBase64String("SGVsbG8="))
